Step 1 - Security of WordPress sites - information of user names

Published on February 28, 2024

Uri Nachmani is a Jetserver information security expert

Risk level: low

This flaw allows a malicious attacker to enumerate user names in various services, with the help of permissive settings in Web applications and with the correct use of brute-force, the malicious attacker can reveal the user name of the management panel. Exposing this sensitive information is a serious loophole because, if the management panel is not protected in any way, the attacker can perform another brute-force attack on the password parameter, thus completing the puzzle necessary to be able to penetrate the site's management area.

Start by updating the package manager cache. If this is the first time you’re using sudo within this session, you’ll be prompted to provide your user’s password to confirm you have the right privileges to manage system packages with apt:

In WordPress systems, there is a depolative definition which, with the help of the author parameter, makes it possible to reveal the names of the users on the website, revealing this information is possible because WordPress associates each author-editor with a certain -id parameter, id number 1 will always be the user with the highest privileges, if this definition is not Blocked ahead of time, you can run - Brute-force of numbers on the author parameter, the server's response will contain the Location header which will reveal the username associated with that id.

During the robustness test, the Nmap tool and the script - http-wordpress-users were used, this script performed a brute-force of numbers while transferring the author parameter to the site, the scan revealed the main user on the site - root, who is associated with id number 1.

A manual test can be done By accessing the address: http://3.120.144.6:31120/?author=1 and checking the location title returned from the server.

Note: Enabling this feature is something of a judgment call. If enabled, passwords which don’t match the specified criteria will be rejected by MySQL with an error. It is safe to leave validation disabled, collection of guides but you should always use strong, unique passwords for database credentials.

Step 3 - Recommendations for repair

The option that allows the user's names to be revealed must be blocked immediately, it is forbidden to allow the transfer of the parameter - author with the values ​​of the numbers indicating the id of the users on the website.

This can be blocked in 2 ways:
  • Changing the Function.php code in the website template
  • Server-level blocking by editing the .htaccess file and using mod_rewrite to set the rule.
An example of defining the security rule in the .htaccess file:

RewriteCond %{QUERY_STRING} ^author=([0-9]*)

RewriteRule .* http://example.com/? [L,R=302]
